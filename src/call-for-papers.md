## Call for Papers

We invite submissions that study, build, evaluate, or critique trustworthy software engineering agents. VAST 2027 especially welcomes work that treats agent behavior as an inspectable and auditable process rather than only a final generated artifact.

Topics include, but are not limited to:

<div class="compact-topics">

### Verifiable Agentic Software Engineering

- Agent traces, provenance, and workflow observability for software engineering tasks
- Evidence-grounded reasoning over code, tests, logs, issues, documentation, and tool outputs
- Trace-based verification and validation of agent actions, decisions, and generated artifacts
- Reproducibility and auditability of agent-generated code, patches, tests, commands, and deployment steps
- Test-based, specification-based, static-analysis-based, and formal-methods-based checking of agent behavior
- Failure localization and diagnosis from intermediate reasoning traces, tool calls, and execution logs
- Benchmarks and datasets with executable tasks, agent traces, evidence chains, and verifiable outcomes

### Aligned Agentic Software Engineering

- Alignment of SE agents with developer intent, issue descriptions, requirements, and task scope
- Project-aware agents that respect coding style, architecture, API contracts, dependencies, and repository conventions
- Policy-aware agents for organizational rules, release processes, CI constraints, rollback policies, and human approval requirements
- Human-in-the-loop review, oversight, intervention, and delegation mechanisms for agentic SE workflows
- Evaluation of task faithfulness, requirement preservation, behavioral consistency, and unintended side effects
- Developer-centric studies of trust, controllability, usability, and collaboration with SE agents
- Alignment failures in code generation, testing, debugging, repair, maintenance, documentation, DevOps, and security analysis

### Secure Agentic Software Engineering

- Prompt injection, context injection, and adversarial artifacts in code comments, issues, logs, documentation, tests, and web content
- Secure tool use, command execution, file-system access, dependency management, and deployment automation
- Runtime monitoring, guardrails, sandboxing, permission models, and policy enforcement for SE agents
- Red-teaming, stress testing, and adversarial evaluation of agentic SE workflows
- Prevention of credential leakage, unsafe external interactions, destructive actions, and unauthorized modifications
- Supply-chain risks in agent-generated code, dependencies, build scripts, configurations, and CI/CD workflows
- Security analysis agents that are themselves robust against manipulation, tool abuse, and open-world attacks

### Cross-Cutting Topics

- Trustworthy agents for code generation, bug fixing, testing, debugging, program repair, DevOps, maintenance, documentation, and security analysis
- Evaluation metrics beyond final correctness, including verifiability, alignment, security, robustness, and accountability
- Empirical studies, industrial experiences, tools, datasets, infrastructures, and open challenges for trustworthy agentic SE

</div>

## Submission Types

- **Position Papers (1-4 pages including references):** Well-argued position papers or work in progress.
- **Research Papers (4-8 pages including references):** Technical research, experience reports, empirical studies, tools, benchmarks, and artifact-focused work.

## Paper Submission

All submissions must conform to the IEEE conference proceedings template, as specified in the IEEE Conference Proceedings Formatting Guidelines. In particular, submissions should use a 24pt title and 10pt body text. LaTeX users must use `\documentclass[10pt,conference]{IEEEtran}` without the `compsoc` or `compsocconf` options.

Please note that IEEE format is used this year. Submissions must strictly follow the IEEE conference proceedings formatting instructions above. Changes to spacing, font size, margins, or other formatting requirements may result in desk rejection without further review.

Detailed submission policies and formatting guidelines are aligned with the [ICSE 2027 Research Track submission process](https://conf.researchr.org/track/icse-2027/icse-2027-research-track).

## Review Process

VAST 2027 will follow a double-blind peer-review process. Authors must anonymize their submissions and avoid revealing their identities, affiliations, or prior work in a way that compromises double-blind reviewing.

Each submission will be reviewed by at least three members of the program committee. The review process will be coordinated by the program chairs to ensure fairness, consistency, and relevance to the workshop scope.

Submissions will be evaluated based on the following criteria:

- **Novelty:** The originality of the problem formulation, technical approach, methodology, empirical findings, theoretical insight, artifact, or position presented in the paper.
- **Rigor:** The soundness, clarity, and depth of the contribution, including the appropriateness of the methods, analysis, evaluation design, and supporting evidence.
- **Relevance:** The significance of the work to trustworthy agentic software engineering, including its connection to verifiability, alignment, security, or broader challenges in agentic SE workflows.
- **Verifiability and Transparency:** The extent to which the paper provides sufficient information for readers to understand, assess, and, where applicable, independently verify or reproduce the claimed contributions. This may include clear descriptions of agent workflows, traces, datasets, experimental settings, artifacts, or evaluation protocols.
- **Presentation:** The clarity, organization, and readability of the paper, including how well the motivation, contributions, methodology, results, and limitations are communicated.

Reviewers will consider all criteria holistically. Papers should clearly explain and justify their claimed contributions, and should make explicit how they advance the understanding, development, evaluation, or deployment of verifiable, aligned, and secure SE agents.
